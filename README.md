# Fast Past - Aplicație web pentru organizarea de conferințe

## Specificatii proiect și planul de proiect


### Specificatii proiect    

 _1.1. Titlul proiectului_  
	&nbsp;Titlul proiectului nostru este MindMeet - aplicație web pentru organizarea și gestionarea conferințelor științifice.  

 _1.2. Obiectiv_  
 &nbsp;Obiectivul proiectului nostru este sa realizam o aplicatie web in care se pot organiza si gestiona conferinte stiintifice ce au ca scop colaborarea organizatorilor, autorilor și reviewerilor pentru înscrierea, evaluarea și aprobarea articolelor științifice.  

 _1.3. Descriere generala_  
 &nbsp;Aplicația este dezvoltată sub forma de Single Page Application (SPA), care asigura o experienta de utilizare rapidă, fluidă și responsiva. Va fi complet accesibilă si optimizata pentru desktop, smartphone sau tabletă.  
 &nbsp;Aceasta permite gestionarea completă a fluxului dintr-o conferință: definirea evenimentului, înscrierea autorilor, trimiterea articolelor, alocarea automată a reviewerilor și gestionarea feedback-ului.  

 _1.4. Funcționalități cheie_    
 
&nbsp;**General**  
&ensp;-Autentificare/ înregistrare  
&ensp;-Resetare parola  
&ensp;-Selectare rol  
&ensp;-Gestionarea profilului utilizatorului    
	
&nbsp;**Organizator**  
&ensp;-Creeaza o conferință  
&ensp;-Configurează datele evenimentului (titlu, descriere, termene)  
&ensp;-Alocă manual sau automat rewierii disponibili  
&ensp;-Monitorizează progresul articolelor  
&ensp;-Confirmă acceptarea/respingerea articolelor evaluate    
	
&nbsp;**Autor**  
&ensp;-Se înregistrează în aplicație  
&ensp;-Selectează conferința la care dorește să participe  
&ensp;-Încarcă un articol (document PDF + metadate)  
&ensp;-Vizualizează feedback-ul primit  
&ensp;-Poate trimite versiuni actualizate ale articolului    
	
&nbsp;**Reviewer**  
&ensp;-Primește automat articole pentru evaluare (2 revieweri/articol)  
&ensp;-Poate aproba articolul sau poate solicita modificări  
&ensp;-Trimite comentarii și recomandări autorului    

### Planul proiectului    

  _2.1. Definirea proiectului_  
&ensp;-Stabilirea obiectivului general al aplicației  
&ensp;-Identificarea tipurilor de utilizatori  
&ensp;-Stabilirea funcționalităților principale și a cerințelor minime    
	
  _2.2. Analiza cerințelor_  
&ensp;-Identificarea cerințelor funcționale (ce trebuie să facă aplicația)  
&ensp;-Analiza fluxurilor de lucru: trimiterea articolelor, evaluarea, aprobarea  
&ensp;-Stabilirea arhitecturii generale (SPA, API, baze de date)    
	
  _2.3.Designul aplicației_  
&ensp;-Structurarea componentelor aplicației (frontend + backend)  
&ensp;-Planificarea structurii bazei de date  
&ensp;-Proiectarea interfeței utilizator    
	
  _2.4.Implementarea_  
&nbsp;**Backend**  
&ensp;-Implementarea sistemului de autentificare și gestionare roluri  
&ensp;-Crearea API-urilor pentru conferințe, articole și revieweri  
&ensp;-Implementarea algoritmului de alocare a reviewerilor  
&ensp;-Gestionarea încărcării fișierelor  
&nbsp;**Frontend**  
&ensp;-Implementarea interfeței pentru fiecare tip de utilizator  
&ensp;-Realizarea componentelor UI și a formularelor  
&ensp;-Gestionarea navigației și comunicării cu backend-ul  
&ensp;-Asigurarea compatibilității pe desktop și mobil    
	
  _2.5. Testarea_  
&ensp;-Testarea funcționalităților individuale  
&ensp;-Testarea interacțiunilor între frontend și backend  
&ensp;-Validarea încărcării fișierelor și a procesului de evaluare  
&ensp;-Bug fixes    
	
  _2.6.Implementarea finală și livrarea_  
&ensp;-Integrarea componentei frontend cu backend-ul  
&ensp;-Optimizarea performanței și experienței utilizatorului  
&ensp;-Verificarea completă a tuturor funcționalităților  
&ensp;-Pregătirea unei versiuni stabile a aplicației
